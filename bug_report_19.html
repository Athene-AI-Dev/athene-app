BUG-19

File Search Shows '0 of 4 Files' — Table Remains Empty

Area Upload / Assets
Severity HIGH
Status Fixed

Issue Description

When users entered a search query that yielded no matches in the file upload/assets section, the UI indicated "Showing 0 of 4 files" in the summary area. However, the table itself was entirely blank, displaying no rows or empty state. This lack of visual feedback made the file management feature appear broken to the end-users.

Root Cause

In the app/(dashboard)/files/page.tsx file, the TableBody only mapped over the filteredFiles array. When the search produced zero results, the array length was 0, which resulted in the map function returning an empty output without a fallback empty state.

Resolution

Added an explicit conditional check to render an empty state row when filteredFiles.length === 0. This row spans all columns and clearly displays a "No results found" message along with a search icon to guide the user.

Code Implementation

// Before
) : (
  filteredFiles.map((file) => ( ... ))
)

// After
) : filteredFiles.length === 0 ? (
  <TableRow className="border-white/5">
    <TableCell colSpan={5} className="py-24 text-center">
      <div className="flex flex-col items-center justify-center space-y-3">
        <Search className="w-10 h-10 text-slate-600" />
        <p className="text-[14px] font-bold text-white">No results found</p>
        <p className="text-[12px] font-medium text-slate-500">We couldn't find any files matching your search.</p>
      </div>
    </TableCell>
  </TableRow>
) : (
  filteredFiles.map((file) => ( ... ))
)

Success: The application now gracefully handles empty search results, preserving the table structure and explicitly notifying the user.
