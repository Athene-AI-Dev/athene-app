BUG-20

Intelligence Layers Show 0 Files Despite Uploads

Area Assets / Intelligence Layers
Severity MEDIUM
Status Fixed

Issue Description

Intelligence layers (Financial Records, Legal Discovery, etc.) displayed zero files despite multiple files existing in the system. The file layer distribution counters were non-functional, making it appear that files were not categorised or the pipeline was disconnected.

Root Cause

The file categorization pipeline was not fully connected. The file upload and fetch routes (app/api/files/route.ts and app/api/files/upload/route.ts) were hardcoding every file's layer to 'Internal Wiki', ignoring the metadata and file content. Additionally, the optimistic UI update in app/(dashboard)/files/page.tsx explicitly overrode the layer to "Internal Wiki", meaning no files were ever categorized into Financial Records, Legal Discovery, or Audit Logs.

Resolution

We implemented an auto-categorization pipeline that evaluates file names and types upon upload and fetch. The pipeline detects keywords (like "invoice", "contract", "audit") and assigns files to the corresponding intelligence layers automatically.

Modifications Made

  • app/api/files/route.ts: Modified the GET response to derive the layer based on the filename/type if a custom layer isn't stored in metadata.
  • app/api/files/upload/route.ts: Updated the POST endpoint to deduce the appropriate intelligence layer during file upload and save it persistently to metadata.layer in Supabase.
  • app/(dashboard)/files/page.tsx: Updated the optimistic UI state to apply the same classification rules instantly so the user sees the layer accurately before the backend response returns. The hardcoded layer: "Internal Wiki" override was also removed from the completed request map.

Code Implementation

// Dynamic categorization pipeline (added to api routes and UI)
const nameLower = file.name.toLowerCase();
let layer = 'Internal Wiki';

if (nameLower.includes('invoice') || nameLower.includes('financial') || nameLower.includes('tax') || ext === 'XLSX' || ext === 'CSV') {
    layer = 'Financial Records';
} else if (nameLower.includes('contract') || nameLower.includes('legal') || nameLower.includes('nda') || nameLower.includes('agreement')) {
    layer = 'Legal Discovery';
} else if (nameLower.includes('audit') || nameLower.includes('log')) {
    layer = 'Audit Logs';
}

// Ensure the layer is saved securely
const { data: doc } = await supabaseAdmin.from("documents").insert({
    ...
    metadata: { size: sizeStr, type: ext, uploaded_by: ownerId, layer },
});

Success: Files are now dynamically and accurately categorised upon upload. The Intelligence Layer sidebar counts have been fully restored and dynamically reflect the actual repository status.
